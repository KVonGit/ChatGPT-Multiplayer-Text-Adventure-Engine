Multiplayer Text Adventure Engine - Documentation

Overview

This engine is a lightweight multiplayer text adventure system that uses PHP, JavaScript, and JSON for game state storage. It allows players to interact with the game world through text-based commands.

File Structure

  • index.php - Handles player login and initializes the game.

  • game.php - Main game interface where players enter commands.

  • command.php - Processes player commands and updates game state.

  • save_game.php - Saves game state to a JSON file.

  • core.js - Contains the game engine logic.

  • game_state.json - Stores the game world and player data.

Setting Up the Game

  1. Upload all files to an Apache server with PHP enabled.

  2. Ensure game_state.json exists in the directory (it will be auto-created if missing).

  3. Open index.php in a browser and enter a player name.

  4. Interact with the game in game.php by entering text commands.

Creating a Custom Game World

  1. Open core.js and define locations and objects as JavaScript objects.

  2. Add interactions and logic to command.php (e.g., handling new commands).

  3. Modify game_state.json manually or through gameplay to prepopulate world state.

Example Command Handling

  • pickup [item] - Adds an item to the player’s inventory.

  • move [location] - Can be implemented to change player locations.

  • use [item] - Custom actions based on inventory items.

Expanding the Game

  • Add more interactions in command.php.

  • Enhance core.js with more game mechanics.

  • Implement WebSockets for real-time updates instead of AJAX polling.

Troubleshooting

  • Ensure PHP is enabled on the server.

  • Check game_state.json for errors if game state isn’t updating.

  • Use browser console logs and error_log in PHP for debugging.

This documentation provides a starting point for customizing and expanding the multiplayer text adventure engine. Happy coding!
